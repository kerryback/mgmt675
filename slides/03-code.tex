\begin{frame}{Benefit of Bundling Code Execution with AI}
\begin{baritemize}
  \item LLMs can write code, but writing is not the same as running
  \item Attaching a code execution tool enables:
  \begin{itemize}
    \item Data analysis with real calculations
    \item Visualizations and charts
    \item File processing (Excel, CSV, PDF)
    \item \alert{Iterative debugging---run, fix, repeat}
  \end{itemize}
  \item Transforms chatbots into computational tools
\end{baritemize}
\end{frame}

\begin{frame}{ChatGPT Code Interpreter}
\begin{shadedbox}\centerline{\large What is it?}\end{shadedbox}

\begin{baritemize}
  \item A Python environment that runs inside ChatGPT
  \item ChatGPT writes code, executes it, and returns results
  \item If code fails, it reads error messages and fixes them
  \item Sandboxed environment: secure but limited
\end{baritemize}
\end{frame}

\begin{frame}{Accessing Code Interpreter}
\begin{baritemize}
  \item Available on free tier (limited to twice per day)
  \item ChatGPT Plus (\$20/month) provides more usage
  \item GPT-5 is now the default model (GPT-4o still available)
  \item Code Interpreter is enabled automatically
  \item Upload files directly to the conversation
\end{baritemize}
\end{frame}

\begin{frame}{ChatGPT: What You Can Do}

\begin{baritemize}
  \item Upload CSV, Excel files
  \item Statistical analysis
  \item Financial analysis
  \item Create visualizations
  \item Download results
\end{baritemize}

\end{frame}



\begin{frame}{ChatGPT: Limitations}
\begin{baritemize}
  \item \alert{No internet access}---cannot fetch live data
  \item Maximum file upload: 512 MB
  \item Runtime limit: 120 seconds per execution
  \item Session state clears when environment resets
  \item Pre-installed packages only
\end{baritemize}
\end{frame}

\begin{frame}{ChatGPT Exercise}


\textit{``I'm uploading an Excel file with monthly returns for WMT, the excess market return (MKT-RF) and the risk-free rate (RF).  Calculate excess returns for WMT and regress on MKT-RF to estimate the beta of WMT.  Plot the data and the regression line.''}

\vspace{0.3cm}
ChatGPT will write Python code, run it, show results, and let you download the chart.

\vspace{0.3cm}
\begin{shadedbox}
\centerline{\href{https://mgmt675.kerryback.com/files/betas.xlsx}{Download Data for Beta Exercise}}
\end{shadedbox}
\end{frame}

\begin{frame}{What Do We Need to Check?}

  \begin{baritemize}
  \item Mostly, that it used the correct left-hand side and right-hand variables.  Miscommunication could possibly cause an error.
  \item AI knows how to run regressions with Python, so that is not an issue.
  \item Can ask to see the code.  Python is mostly readable.  If something is unclear, ask ChatGPT about it.
  \item Can check against calculation by other method, but then no need to use AI, unless we are taking a step towards building an app.
  \end{baritemize}
\end{frame}

\begin{frame}{Claude Artifacts}

\begin{baritemize}
  \item Interactive content in a panel next to the chat
  \item Codes in React/JavaScript---runs in your browser
  \item View, edit, and iterate in real-time
  \item \textbf{Save} artifacts to your account for later use

  \item Create and check once.  Use forever!
\end{baritemize}
\end{frame}

 \begin{frame}{Claude Artifact Exercise}

\vspace{0.3cm}

\end{frame}

\begin{frame}{Claude Artifact Limitations}
\begin{baritemize}
  \item Artifacts use JavaScript only (not Python)
  \item Limited libraries
  \item No direct file system access
  \item Uploaded files consume context window
\end{baritemize}
\end{frame}

\begin{frame}{Local Execution: Advantages}
\begin{baritemize}
  \item \textbf{No sandboxing}---access databases, APIs, local files
  \item \textbf{Install anything}---pip install, npm, system packages
  \item \textbf{Large datasets}---no upload limits
  \item \textbf{Persistent state}---pick up where you left off
  \item \textbf{Version control}---code saved in your repo
  \item \textbf{Production path}---code is ready to deploy
\end{baritemize}
\end{frame}
