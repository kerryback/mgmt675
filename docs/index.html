Course Description

Each of us will, in the future if not already, use large language models (LLMs) as assistants. The models and interfaces will evolve, but it is worthwhile to learn what is available now and adapt as things evolve. Undoubtedly, everyone has already used ChatGPT and other LLMs for some purposes. What we will explore in this course is using ChatGPT with python to perform data analysis, specifically financial data analysis. We will use Julius.AI, which is an interface to ChatGPT with python (I’ll call it ChatGPT + Python).

At the end of the course, I anticipate that everyone may still prefer to use Excel for most things. However, there are some things for which ChatGPT + Python is better suited, and there are other things for which ChatGPT + Python may be faster. In any case, it is worthwhile to learn what can be done with this new tool, so we will be better prepared for the future. It is quite possible that everyone will end up using Python in Excel and Microsoft Copilot, rather than Julius.AI, but Python in Excel is not generally available yet, and, in any case, Microsoft Copilot + Python in Excel will also be a version of ChatGPT + Python. It is also quite possible that Google or someone else may surpass (or may already have surpassed) OpenAI, so maybe we should say LLM + Python, but we will use ChatGPT in this course.

The format of the course is that we will successively consider different types of analyses. We will perform each analysis live in class using Julius.AI. Then, your assignments will be to have similar conversations and to produce a figure or table or report, depending on the particular assignment.

We will spend roughly the first quarter of the course on topics that were covered in Applied Finance. This course will differ from Applied in that we will ask ChatGPT + Python to do the work for us, instead of using Excel and @Risk. In the remaining 3/4 of the course, we will cover new topics, including machine learning and forecasting.

Topics

  1. Intro and simulation
  2. Data handling
  3. Cost of capital and performance evaluation
  4. Visualization
  5. Portfolio optimization
  6. Return prediction by machine learning
  7. Classification by machine learning
  8. Forecasting

Software

Julius.AI provides a 50% academic discount. Sign up for a free account, then send an email using your Rice email account to team@julius.ai and ask for the academic discount. They will respond with a promo code to use. The Basic account allows 250 messages per month and will probably be ok. If the message limit becomes binding, you can always switch at that time to the Essential account, which allows unlimited messages. By using Julius, we can upload data files for ChatGPT to read and get ChatGPT to download and work with data from the web. You don’t need a separate account for ChatGPT through OpenAI. The Julius account will give you access to ChatGPT. Everything is run in the cloud from a web browser, so there is no software to download.

In addition to a Julius account, you should have a Google account for Google Colab and Google Drive. You won’t need any other software for the course. In particular, it is not necessary to have python installed on your computer.
