Claude Desktop Overview

Three Tabs: Chat, Cowork, Code

Chat (Analysis)

  • Sandboxed Python in the browser
  • No local file access
  • Must upload data manually

Cowork

  • Runs in a local VM
  • Files synced to/from VM
  • No internet access

Code

  • Runs on your machine
  • Full local file access
  • Full internet access

Code mode gives Claude direct access to your machine — no sandbox, no VM, no upload step.

Which Tool When?

  • Chat for questions, explanations, and brainstorming
  • Artifacts for quick interactive tools (e.g., a DCF calculator with sliders)
  • Code can do everything Cowork can do, plus it has full internet access and direct access to your local files
  • Code can also build and deploy interactive tools (like Artifacts)
  • Cowork’s advantage is that it requires no local software setup and runs in an isolated sandbox

Class Installers

The class installer sets up everything you need in one step:

  • Python — language Claude Code uses to execute analysis
  • Claude Desktop — the application with built-in Claude Code
  • Skills — pre-packaged instructions for Excel, Word, PowerPoint, and more

Running the Installers

Windows

  1. Download Windows Installer
  2. Extract the zip file
  3. Right-click INSTALL.bat → Run as administrator

Mac

  1. Download Mac Installer to Downloads
  2. Open Terminal
  3. Run: cd ~/Downloads && bash install.sh

Getting Started

  1. Open Claude Desktop → Code tab
  2. Select a working folder for your project
  3. Choose a model: Sonnet (fast) or Opus (thorough)
  4. Set permission mode to auto-accept (recommended for exercises)

CLAUDE.md

Claude reads CLAUDE.md files automatically at the start of every session — no need to repeat instructions.

  • Global (~/.claude/CLAUDE.md): applies to every project — output preferences, preferred libraries, personal conventions
  • Per-Project (project root): applies only to this folder — data context, domain guidance, project-specific workflows

The installer pre-configured skills for Excel, Word, and PowerPoint. Prompt Claude Code to document them in your global CLAUDE.md:

“Read the skills in my .claude/skills folder and add a summary of each to my global CLAUDE.md.”

Once documented, Claude will automatically follow those instructions without needing slash commands.

Example

“Create an Excel workbook with a loan amortization table for a $300,000 mortgage at 6.5% annual interest with a 30-year term and monthly payments. Use formulas so I can change the loan amount, rate, and term. Include columns for payment number, payment, principal, interest, and remaining balance.”

Claude Code in the Terminal and VS Code

Terminal Alternative

The Code tab in Claude Desktop and the terminal command claude use the same engine.

Claude Desktop (Code tab)

  • Visual interface with file browser
  • Click to select working folder
  • Good for getting started

Terminal (claude)

  • Type claude in any terminal
  • Same skills, same config
  • Preferred by power users

Both approaches share the same skills, MCP servers, and configuration files.

Installing the Terminal and VS Code

Claude Code (Terminal)

  • Mac/Linux: Open Terminal and run: curl -fsSL https://claude.ai/install.sh | bash
  • Windows: Open PowerShell and run: irm https://claude.ai/install.ps1 | iex
  • Windows also requires Git for Windows
  • Verify with claude --version

VS Code + Extensions

  1. Download VS Code and install
  2. Open Extensions sidebar (Ctrl+Shift+X on Windows, Cmd+Shift+X on Mac)
  3. Search Python (by Microsoft) → Install
  4. Search Claude Code (by Anthropic) → Install

Claude Code updates automatically. The VS Code extension uses the same engine as the terminal — install either or both.

Using Claude Code in the Terminal and VS Code

Terminal

  • cd into your project folder, then type claude
  • First launch: sign in with your Anthropic account
  • Chat naturally — Claude reads and writes files in that folder
  • Type /help for commands, /model to switch models
  • Press Esc to interrupt, Ctrl+C twice to exit

VS Code

  • Open your project folder in VS Code
  • Click the Claude Code icon in the sidebar (or Ctrl+Shift+P → “Claude Code: Open”)
  • Sign in on first use
  • Claude can see your open files, file explorer, and terminal
  • Code changes appear as inline diffs you can accept or reject

Both share the same skills and configuration. VS Code adds a file explorer and visual diffs; the terminal is lightweight and fast.

Working with Multiple Files

Merging Data from Different Sources

Real financial data rarely lives in a single file or format. Claude Code can read CSV, Excel, PDF, and Word files, then combine them into a unified dataset.

Common scenarios:

  • Holdings from multiple brokerages (xlsx + csv)
  • Loan tape (CSV) + appraisal report (PDF) + policy memo (DOCX)
  • Quarterly reports from divisions with different column names

What Claude Handles

  • Reads each file format natively
  • Reconciles column names and data types
  • Flags mismatches and missing values
  • Outputs a clean, unified table

You describe what should be merged and how. Claude handles the parsing and code.

Filtering and Aggregating

Once data is combined, the next step is usually filtering, sorting, or grouping.

Filtering

  • “Keep only loans above $1M”
  • “Filter to stocks with P/E > 0”
  • “Exclude rows with missing revenue”

Aggregating

  • “Average revenue by sector and quarter”
  • “Sum holdings by asset class”
  • “Rank funds by Sharpe ratio”

State your intent in plain English. Claude writes the pandas code, runs it, and shows you the result. Always check the filter logic — this is where silent errors happen most often (Module 7).

Example

Download aggregation.zip and extract the workbooks into a folder.

Each Excel file contains a data table. Column names vary across files and some columns are missing. Ask Claude Code to:

  1. Summarize the columns present, row count, and basic statistics for each file
  2. Combine all files into a single table, reconciling the varying column names
  3. Save the result as combined.xlsx

This is the same “divide and conquer” pattern used in professional data pipelines — read each source independently, then unify.
