Where Are Agents Headed?

Recap: What We’ve Built in This Course

We’ve used AI as a tool — chatbots, skills, agents with tool use, RAG, sentiment analysis.

All of that sits on top of a foundation:

  • A model (Claude, GPT, Gemini) that reads and writes text
  • Tools that let the model take actions (APIs, databases, code execution)
  • A loop that keeps the model running until it’s done

That foundation is called the harness. And as of April 2026, the harness has become the main battleground in AI.

Understanding this landscape will help you evaluate AI products, vendors, and strategies — whether you’re building, buying, or advising.

What Is a Harness?

Martin Fowler defines harness engineering as everything surrounding an AI model, except the model itself:

Orchestration

  • Model invocation
  • Tool selection and routing
  • Multi-step reasoning loops
  • Multi-agent delegation

Infrastructure

  • Sandboxed code execution
  • Session state and memory
  • Error recovery
  • Persistent storage

Governance

  • Scoped permissions
  • Audit trails and tracing
  • Compliance controls
  • Data residency

For 18 months, teams built this themselves using open-source frameworks or custom code. Now the AI labs are shipping it as a product.

The Harness Pricing War

Five Labs, Four Business Models

As of April 2026, every major AI lab has shipped its own harness — but they disagree on pricing:

Provider Product Pricing Model
Anthropic Managed Agents $0.08/session-hour + tokens
OpenAI Agents SDK (open source) Tokens only, no runtime fee
Google Vertex AI Agent Engine Per-component consumption
Microsoft Foundry Agent Service Per-model + per-tool consumption
AWS Bedrock AgentCore Runtime primitives (coming)

Anthropic bundles everything into a session-hour fee. OpenAI gives away the harness and charges only for model usage. Google and Microsoft meter individual components. These are fundamentally different bets.

Why This Matters for Your Career

You don’t need to build a harness. But you need to understand the strategic choices:

Managed (Anthropic, Google, Microsoft)

  • Turnkey: deploy agents without operating infrastructure
  • Vendor controls the runtime
  • Predictable pricing per session
  • Data stays on the vendor’s infrastructure

Open Source (OpenAI)

  • You choose where agents run
  • Bring your own compute and storage
  • No runtime fee from the lab
  • Full control over sensitive data

The choice between managed and self-hosted is a business decision, not a technical one. It depends on data sensitivity, cost structure, vendor tolerance, and team capabilities.

The Historical Pattern

We’ve seen this split before in cloud infrastructure:

  • Terraform (open source) coexisted with AWS CloudFormation (managed)
  • Kubernetes (open source) coexisted with EKS, GKE, AKS (managed)

In both cases, open source didn’t kill managed, and managed didn’t kill open source. They serve different buyer profiles.

The same pattern is forming in AI agents: teams that want convenience go managed; teams that want control go open source.

The lesson for evaluating vendors: ask whether they’re selling the runtime, the model, or both — and what lock-in comes with each.

What Happened to LangChain?

The Rise and Fall of Orchestration Frameworks

2023: LangChain solved a real problem

  • Models couldn’t use tools natively
  • No built-in conversation memory
  • No structured output
  • Chaining prompts required glue code
  • LangChain provided the duct tape

2026: The problem was solved upstream

  • Models handle tool use natively
  • Structured output is built into APIs
  • Labs ship their own agent SDKs
  • The SDK is optimized for its own model
  • The generic wrapper adds overhead, not value

LangChain’s value proposition was “write once, swap models freely.” In practice, different models have different strengths and failure modes. Most production systems settle on one model — the switching cost is in prompt engineering, not API syntax.

The Squeeze on Generic Frameworks

The independent orchestration framework is the most exposed archetype in this market shift:

  1. From above: labs ship free, model-native SDKs that work better with their own models
  2. From below: the models themselves now handle what the framework used to do
  3. The pitch erodes: “model-agnostic” flexibility matters less when you’re committed to one or two models anyway

What survives? Startups that differentiate on governance, compliance, vertical depth, or genuine multi-model control — not generic prompt chaining.

If you encounter LangChain in a job or a vendor pitch, this is the context. It’s not that it was bad — it solved a real problem at the right time. The problem just moved.

Wait — Isn’t Claude Code Already a Universal Agent?

You’ve Been Using One All Semester

Claude Code already does the Dobby pattern: you describe what you want in natural language, and it picks tools, runs code, queries data, writes files.

So why do we need agent SDKs and managed runtimes?

Claude Code / Desktop

  • You’re in the loop, approving and steering
  • Runs on your machine, in your session
  • Single user, interactive
  • Perfect for knowledge work: research, analysis, coding, documents
  • This is what most of us use day-to-day

Agent SDKs / Managed Agents

  • Runs autonomously — no one at the keyboard
  • Runs in the cloud, 24/7
  • Serves many users with different permissions
  • Embedded in products (CRM, trading platform, compliance system)
  • Triggered by events, schedules, or other agents

Claude Code is the universal agent for interactive work. The SDK layer is for agents that act on their own — overnight compliance monitoring, auto-triaging customer tickets, or a research agent that runs while you sleep.

The Frontier: Beyond Interactive

The real frontier isn’t “can an agent use tools?” — you already know it can. It’s four things:

  1. Autonomy — agents that act on triggers, schedules, or events without a human at the keyboard. A compliance agent monitors trades overnight and flags violations by morning.

  2. Scale — one agent design serving thousands of users, each with different permissions and data. Notion runs dozens of parallel delegation tasks across its user base.

  3. Embedding — agents wired into products rather than running in a terminal. Sentry’s agent takes a flagged bug and opens a pull request with no human in between.

  4. Agent-to-agent delegation — a planning agent spawns specialist agents, each with scoped tools and permissions, then synthesizes their results.

The line is blurring — Claude Code already has scheduled triggers and background tasks. But for now, the distinction between “interactive assistant” and “autonomous production agent” is where the SDK and harness layer come in.

The Agent as Universal Interface

Karpathy’s Dobby: A Case Study

Andrej Karpathy built an agent called Dobby that replaced six separate smart home apps with a single WhatsApp conversation.

Before: The App Archipelago

  • Sonos app for music
  • Lutron app for lights
  • Ecobee app for thermostat
  • Shade controller app
  • Pool controller app
  • Security camera app
  • Six logins, six interfaces

After: One Conversation

  • “Dobby it’s sleepy time”
  • Lights off, shades down, thermostat adjusted, music stopped
  • All through WhatsApp
  • Agent discovered devices autonomously
  • Reverse-engineered undocumented APIs
  • No configuration files needed

The method matters more than the demo. Dobby didn’t use pre-built integrations. It scanned the local network, found devices, and figured out how to control them — like a developer learning a new codebase.

From GUI to LUI

Dobby illustrates a deeper shift: from Graphical User Interface to Language User Interface.

The apps don’t disappear — but the user never touches them. The agent becomes the universal interface layer between people and software.

Interface Collapse

  • One conversation replaces many apps
  • The agent picks the right tool
  • Users describe intent, not actions

Nuance Over Sliders

  • “Too loud for conversation”
  • “Rebalance conservatively for retirement”
  • Natural language captures context GUIs can’t

APIs Over UIs

  • Competitive advantage shifts to API quality
  • Agents consume APIs, not interfaces
  • Robust API > beautiful dashboard

Access Democratized

  • Describe what you want in plain language
  • No need to know which menu or function
  • Junior analyst = senior analyst in tool access

What This Means for Finance

Imagine a wealth manager’s daily workflow:

“Show me clients whose equity allocation exceeds their risk tolerance, pull the last three months of performance for those portfolios, and draft an email to each suggesting a review meeting.”

That sentence touches four separate systems: portfolio analytics, risk management, CRM, and email. Today, that’s four apps. Tomorrow, it’s one agent with four tools.

What changes:

  • Vendor selection shifts from “best UI” to “best API”
  • Competitive moats shift from proprietary interfaces to proprietary data
  • Job skills shift from “knows how to use Bloomberg” to “knows what to ask for”
  • Build vs. buy gets real benchmarks: $0.08/session-hour managed, or free SDK + your own compute

What to Watch Going Forward

Five Questions to Track

  1. Which pricing model wins? Does bundled (Anthropic) beat open source (OpenAI) beat per-component (Google)? Or do all three coexist for different buyers?

  2. Who controls the harness layer? The lab that owns how agents are built and run owns the new interface layer. This is the real platform war.

  3. How fast do agents replace apps? Karpathy’s Dobby is a proof of concept. The question is how quickly enterprises adopt the pattern.

  4. What happens to the middle layer? Independent frameworks (LangChain) and startups — do they differentiate into governance and compliance, or get squeezed out?

  5. Where does the value accrue? If agents become the universal interface, the value shifts from UI design to data quality, API robustness, and domain expertise.

These questions will shape AI strategy in finance for the next several years. The technology is moving fast; the business models are still forming.

Key Takeaways

  1. The harness — the control layer around the model — is now a product being sold by every major AI lab

  2. The labs disagree on pricing: managed runtime vs. open-source SDK vs. metered components

  3. Generic orchestration frameworks like LangChain are being squeezed from above and below

  4. Agents are becoming the universal interface between people and software (GUI to LUI)

  5. For finance: the competitive advantage shifts from best interface to best data and best APIs

  6. The strategic question isn’t “should we use AI?” — it’s “who controls the agent layer we’ll depend on?”

You don’t need to build this infrastructure. But you need to understand the landscape well enough to make good vendor, platform, and career decisions as it evolves.
